# AgentBPF

### Abstract
Agentic AI systems—autonomous software agents that iteratively observe their environment, plan actions, and execute tools—are rapidly moving from research prototypes to production deployments.Yet the security and governance infrastructure for the sesystems remains fragmented: existing work addresses point problems such as prompt injection detection, sandboxing, or access control, but does not define a unified runtime enforcement architecture for autonomous tool use. We argue that the critical missing boundary in agentic systems lies between language-model reasoning and tool execution. We present AgentBPF, a conceptual framework and reference architecture for runtime
policy enforcement in agentic systems, drawing an explicit architectural parallel with eBPF’s role in Linux kernel security. AgentBPF defines a programmable control plane at the tool-call boundary, composed of four operational primitives: the Tool Invocation Intercept (tii), Context Taint Tracker (ctt), Semantic Intent Classifier (sic), and Agent Audit Ring Buffer (aarb). We further introduce the Capability Manifest, a declarative least-privilege policy interface, and the Autonomous Action Budget, an operational bound on unchecked autonomous execution before human checkpointing. The contribution of this paper is architectural rather than algorithmic. We define the enforcement model, policy interfaces, and runtime abstractions required to make tool-using agents governable in production; position them relative to concurrent work in agent safety, taint analysis, and systems security; and identify the principal implementation and research questions needed to realize this design in practice.
